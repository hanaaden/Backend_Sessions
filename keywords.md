# PRIMARY KEY vs UNIQUE 
primary key is combination of unique constraints and NOT NULL constraints 
it ensures that the colums identify each row in the table and does not allowed NULL values
## Primary key
Used to serve as a unique identifier for each row in a table.

### Unique key 
Uniquely determines a row that isn't the primary key.

Cannot accept NULL values.
Can accept NULL values.

Only one primary key accepted while multiple unique key are okay
